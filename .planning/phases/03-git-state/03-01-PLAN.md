## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| DB → git2 open | Paths from SQLite repos table are passed to Repository::open — must be canonicalized |
| User filesystem → git2 | Repo paths may contain symlinks or traversal sequences if DB was tampered with |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | infra/git.rs open_and_query — path from DB to Repository::open | mitigate | Canonicalize path via std::path::Path::canonicalize before Repository::open; existing Phase 1 D-13 pattern; implemented in open_and_query |
| T-03-02 | Information Disclosure | git2 Repository::open following symlinks | accept | Paths are always stored as canonical in DB (D-13); symlink following only reaches directories the user already owns; local-only threat surface |
| T-03-03 | Denial of Service | vendored-libgit2 compile time | accept | 15-30s clean build only; incremental builds fast; acceptable for local dev tool |
| T-03-SC | Tampering | cargo add git2/rayon/humantime | mitigate | Blocked by Task 1 checkpoint:human-verify for all [ASSUMED] packages before any Cargo.toml edit |
</threat_model>
