<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller path → refresh_git_state | Path argument from any caller (tray, CLI) must be safe before Repository::open |
| rayon thread → git2 | Each rayon thread opens its own Repository — no shared mutable state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Tampering | services/git_state.rs refresh_git_state — path argument | mitigate | Canonicalize path before delegating to open_and_query; double canonicalization is harmless |
| T-03-05 | Tampering | rayon threads sharing git2 state | accept | Architecture ensures one Repository per thread (into_par_iter creates new closure scope per item); Repository is Send not Sync — compiler prevents sharing |
| T-03-06 | Information Disclosure | git_state_error field contents | accept | Error messages come from git2 (file paths, error codes); no secrets; local use only |
</threat_model>
