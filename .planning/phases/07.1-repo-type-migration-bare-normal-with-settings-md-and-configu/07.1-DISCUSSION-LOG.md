## Path template resolution

| Option | Description | Selected |
|--------|-------------|----------|
| Folder name | `{project}` = directory basename | ✓ (default) |
| Alias if set | `{project}` = alias when configured | ✓ (via setting) |
| Parent + name | Relative path from root | |
| Example A templates | `{project}/bare.git` + `{project}/wtrees/{worktree}` | ✓ |
| Example B templates | `{project}.git` + `{project}.{worktree}` | |
| Global only | Templates in `config.toml` only | ✓ |
| Slash → dot only | Branch sanitization | |
| Hash on collision | Sanitize + short hash suffix | ✓ |

**User's choice:** Setting for `folder_name` vs `alias` (default folder name); Example A defaults; global templates; hash suffix on collision.
**Notes:** User free-text on q1: "on settings with the option of alias or folder name, by default assume folder name"

## Sync preflight gate

| Option | Description | Selected |
|--------|-------------|----------|
| All worktrees clean | Dirty check across every linked worktree | ✓ |
| Strict unpushed | All local branches have upstream and ahead==0 | ✓ |
| Block stash, ignore submodules | Stash blocks; submodules N/A v1 | ✓ |
| Default branch target | bare→normal uses main/master worktree | ✓ |

**User's choice:** Custom strict unpushed rule (all branches, all upstreams, all ahead==0); all-worktree dirty; block stashes; bare→normal always default branch.
**Notes:** Stricter than ROADMAP wording ("no unpushed commits" on current branch only).
