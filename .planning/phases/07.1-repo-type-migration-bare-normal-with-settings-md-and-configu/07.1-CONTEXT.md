# Phase 07.1: Repo type migration bare↔normal with SETTINGS.md and configurable path templates - Context

**Gathered:** 2026-06-10
**Status:** Ready for planning

<domain>
## Phase Boundary

Let users migrate a project between **normal git checkout** and **bare + worktree** layouts using configurable path templates, documented in `SETTINGS.md`, with conversion allowed only when the repo passes a strict sync preflight. Conversion flow: rename original directory to `{original}{temp_suffix}` → fresh local clone into target layout → minimal health check → update Workpot catalog path (optional filesystem delete of `.temp` per setting).

Does **not** include tray convert UI, submodule migration, or automatic layout discovery beyond existing watch roots.

</domain>

<decisions>
## Implementation Decisions

### Path template resolution
- **D-01:** `{project}` resolves from a global setting `migration.project_name_source`: `folder_name` (default) or `alias`. When `alias` is selected but the repo has no alias, fall back to folder name.
- **D-02:** Default bare layout templates (global, in `config.toml`): bare repo `{project}/bare.git`, worktree `{project}/wtrees/{worktree}` (ROADMAP example A).
- **D-03:** Path templates are **global only** in `config.toml` — no per-repo template overrides in v1.
- **D-04:** `{worktree}` sanitization: map `/` → `.` in branch names; on path collision after sanitization, append a short hash suffix to disambiguate.

### Sync preflight gate
- **D-05:** “Clean” means **no dirty state in any linked worktree** — run git status semantics across all worktrees (single worktree for normal; all worktrees under bare for bare repos).
- **D-06:** “Fully synced” is strict: **every local branch** must have an upstream configured **and** `ahead == 0` on each branch (not only the current branch).
- **D-07:** Block conversion if **any stash** exists. **Submodules ignored** for v1 preflight.
- **D-08:** **Bare → normal:** target checkout is always the **default branch** (`main` / `master` detection), not necessarily the worktree path the user invoked convert on.

### Conversion clone strategy
- **D-09:** Fresh clone source is the **renamed local `.temp` directory** (`git clone` from local path) — no network required.
- **D-10:** **Normal → bare:** first (only initial) worktree uses the **currently checked-out branch** at convert time.
- **D-11:** Health check is **minimal:** `git rev-parse HEAD` succeeds; for bare layout, `git worktree list` shows expected paths.
- **D-12:** On clone or health-check failure: **leave `.temp` rename in place**, abort, **do not update DB** — user recovers manually.

### Command surface & SETTINGS.md
- **D-13:** CLI: `workpot repo convert <path> --to bare|normal` with clear preflight error messages per failed gate.
- **D-14:** `SETTINGS.md` at **workpot repo root** — user-facing documentation of all settings (including migration templates, temp suffix, delete-original, project name source) with ROADMAP examples.
- **D-15:** **CLI only** for v1 — no tray convert button or detail-pane action in this phase.
- **D-16:** Support `--dry-run`: print resolved target paths and preflight gate results; no filesystem or DB changes.

### ROADMAP-locked (not re-litigated)
- **D-17:** `migration.temp_suffix` default `.temp` (always applied on rename; not optional).
- **D-18:** `migration.delete_original` default `false` — when true, delete renamed `.temp` after successful conversion; **DB always drops old path** regardless of filesystem delete.
- **D-19:** Bare → normal target path is `{project}` only (no worktree template segment).

### Claude's Discretion
- Exact config.toml key nesting (`[migration]` table shape), hash length for collision suffix, and default-branch detection order when both `main` and `master` absent.
- Whether `--dry-run` prints human table vs JSON (default human).
- Post-success catalog update: re-use `register_manual` / path-key update vs dedicated migration service (must preserve alias, tags, notes, pin).

</decisions>

<specifics>
## Specific Ideas

- User wants **strict sync**: all worktrees clean, all branches pushed with upstream, no stashes — aligns with “safe conversion only when fully in sync” in ROADMAP.
- Path layout preference: nested `{project}/bare.git` + `{project}/wtrees/{worktree}` (not flat `.git` sibling style).
- Collision handling via hash suffix on `{worktree}` path — avoids silent overwrites when branch names sanitize identically.

</specifics>

- Per-repo path template overrides — global config only for v1 (D-03).
