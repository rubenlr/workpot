    ## Repo Migration Settings (`[migration]`)
    This section must include:

    ### Keys and defaults
    - `migration.temp_suffix` (default `.temp`) — suffix appended to the original directory name during conversion; always applied; must not be empty
    - `migration.delete_original` (default `false`) — when `true`, the renamed `.temp` directory is deleted after successful conversion; when `false` (default), it is kept for manual inspection
    - `migration.bare_repo_template` (default `{project}/bare.git`) — path template for the bare git repo; must contain `{project}`
    - `migration.worktree_template` (default `{project}/wtrees/{worktree}`) — path template for the first worktree; must contain `{project}` and `{worktree}`
    - `migration.project_name_source` (default `folder_name`, options: `folder_name` | `alias`) — source for the `{project}` variable; `folder_name` uses the directory name; `alias` uses the workpot alias (falls back to folder name if no alias is set)

    ### Template variables
    Explain: `{project}` is the project name (from `project_name_source`); `{worktree}` is the sanitized branch name (slashes replaced with dots; collision-safe suffix appended if needed). Templates are global only — no per-repo overrides in v1.

    ### Template examples
    Show both ROADMAP examples verbatim:

    **Example A** (nested, recommended): `bare_repo_template = "{project}/bare.git"`, `worktree_template = "{project}/wtrees/{worktree}"`
    - Branch `feature/my-branch` in project `my-project-test` → `my-project-test/bare.git` + `my-project-test/wtrees/feature.my-branch`

    **Example B** (flat sibling): `bare_repo_template = "{project}.git"`, `worktree_template = "{project}.{worktree}"`
    - Same project and branch → `my-project-test.git` + `my-project-test.feature.my-branch`

    Show a minimal config.toml example:
    ```toml
    [migration]
    bare_repo_template = "{project}/bare.git"
    worktree_template = "{project}/wtrees/{worktree}"
    delete_original = false
    temp_suffix = ".temp"
    project_name_source = "folder_name"
    ```

    ### Conversion command
    - `workpot repo convert <path> --to bare` — convert a normal checkout to bare+worktree
    - `workpot repo convert <path> --to normal` — convert a bare+worktree to normal checkout
    - `workpot repo convert <path> --to bare --dry-run` — print resolved paths and preflight results without making changes

    ### Preflight gate
    Conversion is blocked unless: no dirty state in any worktree; all local branches have an upstream and are not ahead; no stash entries. All conditions must be true simultaneously.

    ### Recovery from interrupted conversion
    If the process crashes between rename and clone (or any mid-conversion failure), the original directory is renamed to `{original}{temp_suffix}` (e.g., `my-project.temp`) and the catalog still points to the original path. To recover: either `mv my-project.temp my-project` to restore the original, or `workpot repo add my-project.temp` to register the renamed directory while you decide what to do next. The catalog drops the old path automatically on successful conversion but not on failure.

    Do not use emojis in this file. Use plain markdown headings and code blocks. Prefer code blocks over prose for key/value examples. Keep each setting description to 1-2 sentences.
