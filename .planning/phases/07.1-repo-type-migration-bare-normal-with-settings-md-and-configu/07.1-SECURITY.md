# Phase 07.1 — Security

> Per-phase security contract: threat register, accepted risks, and audit trail.

---

## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| test fixtures → git subprocess | Test helpers invoke `git` CLI via `common::git_cmd()` / `git_cmd_clean()` which strip GIT_* env vars | Repository paths in tempdirs only |
| user path → git2 open | Repo path from caller is canonicalized before passing to git2 | Filesystem paths |
| branch names → stash check | `stash_foreach` requires `&mut repo`; non-UTF8 names blocked at preflight | Branch name strings |
| user-supplied path → git subprocess | `path` arg passed to `git clone` via `Command::arg()`, never shell-interpolated | Filesystem paths |
| path template resolution → filesystem | Resolved paths validated against parent directory to prevent traversal | Derived filesystem paths |
| rename collision → DB state | Pre-check for existing `.temp` before any mutation | Catalog DB rows |
| CLI user input `<path>` → AppContext | Path from CLI arg is a `PathBuf`; passed to `convert_repo` as `&Path`, canonicalized inside service | Filesystem paths |
| ConvertTarget enum → core service | `CliConvertTarget` converted to `workpot_core::ConvertTarget` in match arm; no string interpolation | Enum variants only |
| documentation → user | SETTINGS.md is static prose; no code execution involved | Placeholder example paths |
