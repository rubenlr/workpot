  <behavior>
    `convert_repo(conn, config, path, target, dry_run)`:
    - `dry_run=true`: resolve target paths, run preflight, print resolved paths and gate result, return `Ok(ConvertResult::DryRun { .. })` — no filesystem or DB changes
    - Rejects if path is a git worktree (not bare and not the root): returns `Err(WorkpotError::ConversionPreflight("path is a git worktree; run convert on the bare repo at <git_common_dir>".into()))`
    - Rejects wrong layout: `--to bare` on a bare repo returns `Err(WorkpotError::ConversionPreflight("already bare".into()))`; `--to normal` on a normal repo returns `Err(WorkpotError::ConversionPreflight("already normal".into()))`
    - Preflight gate: calls `run_preflight` — any non-Ready result returns `Err(WorkpotError::ConversionPreflight(<message>))`
    - Pre-checks rename collision: if `{original}{temp_suffix}` already exists, returns `Err(WorkpotError::ConversionPreflight("temp path already exists: <path>; remove it first".into()))` (per T-rename-collision)
    - Rename: `std::fs::rename(original, temp_path)` — on error, returns `Err(WorkpotError::ConversionFailed("rename failed: <e>".into()))` with `log::warn!`
    - Normal→bare clone: `git_cmd_clean().args(["clone", "--bare", "-q", temp_path.to_str()?, bare_git_path.to_str()?])` with `.current_dir(parent)`; then `git_cmd_clean().args(["worktree", "add", worktree_path.to_str()?, &branch]).current_dir(&bare_git_path)` — if any step fails, log warn, return `Err(WorkpotError::ConversionFailed(...))`, leave `.temp` in place, do NOT update DB (D-12)
    - Bare→normal clone: `git_cmd_clean().args(["clone", "-q", "-b", &default_branch, temp_path.to_str()?, target_path.to_str()?])` with `.current_dir(parent)` — same failure behaviour (D-12)
    - Do NOT call `fs::create_dir_all` on clone target directories — let `git clone` create them (per RESEARCH Pitfall 3)
    - Health check (D-11): `git rev-parse HEAD` must succeed in the cloned repo; for bare layout, `git worktree list` must show the expected worktree path — on failure, leave `.temp`, return `Err(WorkpotError::ConversionFailed(...))`, no DB update
    - Catalog path swap: call `catalog_path_swap(conn, old_key, new_key, new_name, new_git_common_dir)` — this resolves `git_common_dir` by calling `crate::infra::git::resolve_git_common_dir(&new_path)` for the new layout
    - `log::info!("convert_repo: {} -> {}", old_path.display(), new_path.display());`
    - After successful DB update, if `config.migration.delete_original == true`: `std::fs::remove_dir_all(&temp_path)` with `log::warn!` on error (non-fatal); if false, leave `.temp` in place
    - Return `Ok(ConvertResult::Converted { from: old_path, to: new_path })`

    `ConvertResult` enum: `Converted { from: PathBuf, to: PathBuf }` and `DryRun { preflight: PreflightResult, resolved_paths: Vec<(String, PathBuf)> }`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user-supplied path → git subprocess | `path` arg is passed to `git clone` via `Command::arg()` calls, never shell-interpolated |
| path template resolution → filesystem | Resolved paths validated against parent directory to prevent traversal |
| rename collision → DB state | Pre-check for existing `.temp` before any mutation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07.1-03-01 | Tampering | Path traversal via template `../` | mitigate | After resolving bare_git_path and worktree_path from templates, assert path starts_with(parent_dir); reject with `ConversionPreflight("resolved path escapes parent directory")` if not |
| T-07.1-03-02 | Tampering | CLI `<path>` arg to git subprocess | mitigate | `git_cmd_clean().args([...]).arg(path)` — path passed as separate `Command::arg()`, never interpolated into shell string; `git_cmd_clean()` strips GIT_* env vars |
| T-07.1-03-03 | Spoofing | Rename collision: convert run twice, `{original}.temp` already exists | mitigate | `if temp_path.exists() { return Err(WorkpotError::ConversionPreflight("temp path already exists: <path>; remove it first")) }` before `fs::rename` |
| T-07.1-03-04 | Tampering | DB updated before health check | mitigate | Strict ordering enforced: rename → clone → health check → `catalog_path_swap` → optional delete; DB update is last before optional FS delete |
| T-07.1-03-SC | Tampering | npm/pip/cargo installs | accept | No new packages introduced |
</threat_model>
