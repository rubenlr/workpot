## Implementation Notes for Planner

### Anti-pattern guard (from RESEARCH.md)

All plans must verify these constraints are met before marking tasks complete:

1. **catalog_path_swap**: Must UPDATE `repo_tags.repo_path` in the same transaction as `UPDATE repos SET path=...`; ON DELETE CASCADE does not fire on PK UPDATE.
2. **stash_foreach**: Requires `&mut Repository`; the preflight function that checks stash must open repo with `mut repo`.
3. **git clone target**: Do NOT call `fs::create_dir_all` on the clone target directory — let `git clone` create it.
4. **Failure path**: On any error after `fs::rename(original → original.temp)`, leave `.temp` in place and do NOT update DB.
5. **Order of operations**: rename → clone → health check → DB UPDATE → optional fs::remove (delete_original).

### `ConvertTarget` enum for CLI

The `--to` flag parses `bare` or `normal`. Implement as:
```rust
#[derive(Clone, Debug, clap::ValueEnum)]
pub enum ConvertTarget { Bare, Normal }
```
Place in `repo_convert.rs` (domain type) and `use` it from `main.rs`.
