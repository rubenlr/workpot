# Phase 06.1 Plan 02: Update command contract summary

**Implemented a production `workpot update` command that resolves latest GitHub release assets, verifies checksums before mutation, and enforces deterministic exit semantics across success/current/network/install/running-app paths.**

## Accomplishments

- Added `update` subcommand in `workpot` CLI with `--only-cli`, `--only-tray`, and `--global`.
- Implemented updater service with presence-based target selection, latest-release metadata fetch, checksum validation, and staged replacement flow.
- Added contract smoke tests for target selection, no-op current behavior, exit taxonomy, and checksum mismatch fail-closed behavior.
- Refactored updater internals into explicit stages (`download_verified_asset`, per-target pipeline functions) without changing external behavior.
