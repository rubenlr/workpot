## Phase Goal (slice 3)

**As a** user glancing at the tray, **I want to** see fresh git state without waiting, **so that** dirty repos are accurate before I switch.

<objective>
Implement non-blocking background git refresh on panel open, event-driven UI update, refresh spinner, Cmd+R shortcut, and dynamic tray icon dirty indicator.

Purpose: Connects Phase 3 git batch to tray UX (D-26..D-28, D-31); satisfies GIT-04 perceived latency in tray context.
Output: refresh_all_git_state API, Tauri async command + events, spinner + tray badge.

**Wave 2 sequencing:** Runs after 04-02 (not parallel). Plan 02 owns filter bar, keyboard nav, and `displayRepos` in `+page.svelte`; this plan only adds refresh state, spinner, and event listeners — do not rewrite filter/selection logic from 04-02.
</objective>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| Mutex deadlock during refresh | medium | Release lock before rayon; re-acquire only for persist (Phase 3 pitfall 6 pattern) |
| UI freeze if refresh sync | high | refresh command must spawn async task and return immediately |
| Event spoofing | low | Events emitted only from Rust backend same app handle |
</threat_model>
