<task type="auto">
  <name>Task 1: macOS runtime resolution for bare cursor program (D-33)</name>
  <files>src-tauri/src/launch.rs</files>
  <action>
    Add resolution in `launch.rs` only — do not change `default_launch_cmd()` in `config.rs` (per D-33 and user constraint).

    1. Add `fn is_unqualified_program(program: &str) -> bool` — true when program has no path separator (not `/custom/cursor`, not `./cursor`).

    2. Add `fn resolve_launch_program(program: &str) -> String`:
       - If not `is_unqualified_program(program)` OR program != `"cursor"`, return `program.to_string()` unchanged (honors D-33 overrides: `/Applications/.../cursor`, `code`, custom IDE, etc.).
       - On `#[cfg(target_os = "macos")]`, iterate known install paths in order; use first path where `std::path::Path::is_file()` is true:
         - `/Applications/Cursor.app/Contents/Resources/app/bin/cursor`
         - `{home}/Applications/Cursor.app/Contents/Resources/app/bin/cursor` (use `directories::BaseDirs` or `std::env::var_os("HOME")` like `paths.rs`)
       - If none exist, return `"cursor"` unchanged so spawn still fails with existing error banner text (D-34).
       - On `#[cfg(not(target_os = "macos"))]`, return program unchanged (Ubuntu CI stays no-op).

    3. In `launch_repo`, after `build_command` returns `(program, args)`, set `let program = resolve_launch_program(&program);` before `Command::new(&program)`.

    4. Do not add crates. Do not invoke `which` or shell out. Minimal diff aligned with existing `launch.rs` style.

    5. Optional one-line module doc comment: default template uses bare `cursor`; macOS tray resolves to Cursor.app bundled CLI when not on PATH; set `launch_cmd` to an absolute path to override.
  </action>
  <verify>
    <automated>cargo test -p workpot-tray launch --offline</automated>
    <automated>cargo build -p workpot-tray --offline</automated>
  </verify>
  <done>
    `launch_repo` spawns resolved absolute path when bare `cursor` and bundled binary exists; qualified paths and non-cursor programs unchanged; default config template string untouched.
  </done>
</task>
